DWARF — SP4 State-Machine Fuzz, 8-hour local soak

Sustained chain-sync sequencing attack on a real local cardano-node · adversary image 0.21.0 · isolated devnet · 2026-06-23 00:39→08:39 UTC
The local endurance counterpart to the Antithesis runs: a real cardano-node (relay2) eclipsed to the DWARF state-machine adversary as its sole peer, fed well-formed chain-sync messages in illegal protocol state/agency continuously for 8 hours. Result: 2,386 illegal-sequence injections across all six scenarios, node and adversary up the entire time (RestartCount 0), 0 node crash markers, 0 Error/Critical, max fault-severity 0. The node-side printf error-formatting artifact reproduced 7,158× — consistent, deterministic, and always non-fatal.

Method

An isolated Docker project (sm_soak, unique container names + network — so it cannot collide with any other running devnet) brings up a real cardano-node as relay2 with the DWARF adversary (ghcr.io/j-gainsec/dwarf-adversary:0.21.0, --state-machine-fuzz) as its sole local-root peer. The adversary drives the raw chain-sync mux channel (bypassing the typed-protocol peer, which would forbid these sequences at compile time) and, per connection, injects one of six illegal sequences — all well-formed CBOR, so the bytes decode and the node’s mini-protocol state machine is what must reject them. The node drops the offending mini-protocol on each violation and reconnects, so the per-connection scenario counter cycles all six continuously. Liveness + node-safety were sampled every 15 minutes for 32 samples.

The node-side finding, characterized at volume

When the node rejects an illegal chain-sync sequence, its unexpected-message error-formatting path throws (from the relay2 tracer, node.json):

ns: Net.Mux.Remote.ExceptionExit   miniProtocolNum: 2 (chain-sync)   dir: InitiatorDir
exception: "printf: bad formatting char 'd'"
msg: "Miniprotocol terminated with exception"   sev: Notice
What 8 hours adds over the smoke test: the artifact is deterministic and bounded — exactly 3 occurrences per injection (7,158 over 2,386), never escalating, never reaching Error/Critical, never destabilising the node (RestartCount stayed 0 for 8h). It is a genuine defect in a rarely-exercised error path — the node’s unexpected-message formatter uses a malformed format string — but it is provably non-fatal. Contrast the byte-level (grammar) adversary, whose corrupted frames the node rejects with a clean DecoderFailure: the decoder error path is robust, the protocol-state error path is not. This is the kind of issue only state-machine (not decoder) fuzzing reaches; worth root-causing / reporting upstream.

Root cause — traced

The error originates in Ouroboros.Network.Protocol.ChainSync.Codec (upstream ouroboros-network-protocols; confirmed present on current main and in the deployed node). The decode fallthrough that reports an unexpected message per protocol state passes four arguments to a printf format string with only three conversions in the StNext/StIntersect branches:

-- CORRECT (SingIdle):  2×%s, 4 args
fail (printf "codecChainSync (%s, %s) unexpected key (%d, %d)"
             (show (activeAgency :: ActiveAgency st)) (show stok) key len)

-- BUG (SingCanAwait / SingMustReply / SingIntersect):  1×%s, 4 args
fail (printf "codecChainSync (%s) unexpected key (%d, %d)"
             (show (activeAgency :: ActiveAgency st)) (show stok) key len)

Text.Printf binds positionally: %s ← show agency (String, ok); %d ← show stok — a String, fed to an integer conversion → errorBadFormat 'd' → printf: bad formatting char 'd'. The format string is simply missing its second %s (the SingIdle branch has it). Isolated to ChainSync — every other protocol codec's fallthrough is internally consistent (BlockFetch/KeepAlive use (%s, %s)+4 args; TxSubmission2/LocalStateQuery use (%s)+3 args). Our scenarios land in StNext/StIntersect (a wrong-state message while the consumer awaits a reply), which is exactly the buggy path; an unexpected message in StIdle would format correctly.

Fix (one char × 3 lines): (%s) → (%s, %s) in the SingCanAwait, SingMustReply, and SingIntersect branches. Full upstream bug report (with the printf arity table, reproduction, and diff) prepared as ouroboros-chainsync-printf-bug-report.md.

Bottom line

Eight hours of continuous chain-sync sequencing attack on a real cardano-node: 2,386 illegal-sequence injections across all six scenarios, zero crashes, zero Error/Critical, node and adversary up the entire time. The state-machine surface is node-safe under sustained attack, and the one node-side artifact it surfaces — a format-string defect in the unexpected-message error path — is now characterized at volume (7,158 reproductions, deterministic, always non-fatal).
