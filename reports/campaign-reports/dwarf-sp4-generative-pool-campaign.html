SP4 — Generative mini-protocol sequencing fuzzer: 8h local soak

Target: Haskell cardano-node, on-wire N2N. Adversary image ghcr.io/j-gainsec/dwarf-adversary:0.23.0. Run 2026-06-29 05:06→13:07Z (8h, 480 1-min samples). Reproduce: --seed 0x73c7035ed442ac28. Raw log: reports/raw/sp4-sm-soak-8h.log.
What this is. The deepening of the earlier shallow ChainSync state-machine fuzz (6 hand-coded illegal sequences — which found the printf format-string bug in ChainSync.Codec) into a generative engine: per connection it walks a legal prefix from the protocol's M3 state-machine model, then injects one illegal departure synthesized from a 6-class taxonomy. The frames are well-formed (they decode); the violation is the protocol state / agency / order / multiplicity. The surface under test is the mini-protocol state machine + mux driver + agency enforcement — not the CBOR decoder. The safe, expected node behaviour is to reject and drop the connection; a hang, crash, mux desync, or an accepted illegal transition would be a finding.

Headline result

Node-safe across 5,629 generative illegal sequences over 8 hours.
  • 0 accepted-illegal — the node rejected every illegal sequence (the conservative accepted-illegal heuristic never tripped).
  • 0 liveness alerts — the node's tip advanced continuously the entire run: block 4 → 501, slot 333 → 29066, syncProgress 100% throughout.
  • 0 crashes / 0 adversary restarts / 0 container exits — all services up the full 8h.
  • No new fatal bug surfaced. This run's value is breadth, not endurance: it demonstrates the node's ChainSync / KeepAlive / TxSubmission2 state machines safely reject the full generated illegal-sequence space, not just the 6 hand-picked cases.

Method

  • Legal scaffold (reused, not rebuilt): the M3 state-machine models (corpora/m3/*-state-machine.json) supply the legal transition graph + real N2N wire-frame alphabet per protocol. They are embedded in the binary (file-embed) and parsed at startup.
  • Synthesized departures: the models carry only happy-paths (all expect:ok); the engine derives the illegal sequences — WrongAgency, OutOfState, PrematureTerminal, PostTerminal, Flood, Duplicate. Each single-frame departure is provably illegal-at-state (QuickCheck-verified, 100 cases).
  • Determinism: every sequence is a pure function of the selector; the run is reproducible from --seed 0x73c7035ed442ac28.
  • Oracle: node liveness (tip advance) sampled each minute — the real win/lose signal. The adversary also emits a conservative, heuristic accepted-illegal exploration signal (reachable-only, never a pass/fail), and per-protocol×class reachability signals (for the Antithesis route).
  • Isolation: ran in a dedicated docker project (sp4soak) on a separate network, never colliding with the live bundle.

Limitations / next increment (labelled, not dropped)

  • BlockFetch (#3) un-exercised on-wire. The engine generates BlockFetch illegal sequences correctly (unit-proven), but the node never opens BlockFetch toward our adversary — BlockFetch starts on-demand only after a peer serves valid headers, and this adversary violates ChainSync immediately. Reaching the node's BlockFetch state machine on-wire requires a serve-valid-then-violate adversary (serve a valid ChainSync header to make the node open BlockFetch, then inject the illegal BlockFetch sequence). Scoped as the next increment.
  • Handshake (#0) deferred. Handshake runs inside the version-negotiation phase (withServerNode) before any application mux channel exists; there is no MiniProtocolNum 0 / raw-mux server in the adversary, so driving illegal handshake frames would mean reimplementing server-node + mux bring-up internals. Deferred to a second increment with this reason recorded.
Net on-wire coverage this run: 3 of 4 wired protocols (ChainSync, KeepAlive, TxSubmission2). The fourth (BlockFetch) and Handshake are the documented next steps.

Throughput redesign (2026-06-29) — adversary-initiated concurrent pool

The original server-mode soak above (5,629 injections / 8h ≈ 12/min) was throttled by (1) artificial 100 ms responder pacing and (2) the node's serial reconnect backoff on one peer link. It was rebuilt as an adversary-initiated concurrent pool: N workers each dial the node, inject a generated illegal sequence, force-close (capped at --sm-conn-ms 750), and reconnect — no pacing. Image 0.25.0.
  • Throughput: ~12/min → ~26,000/min at N=16 (~2,200×); measured up to ~53k/min at higher N (node-clean ceiling tuned to keep relay connections well under the 384 soft-limit).
  • BlockFetch gap CLOSED: as the initiator the adversary opens every protocol itself, so BlockFetch (#3) is now exercised on-wire alongside ChainSync/TxSubmission2/KeepAlive — and it now tests the node's server-side state machines (a new surface vs. the responder-side run above).
  • Connection-pressure finding: at N=384 (the node's connection soft-limit) the relay's chain advance transiently stalled ~5 min then recovered (0 accepted-illegal throughout) — a single peer saturating the connection budget briefly pressures liveness. Soak run at N=16 for an unambiguous liveness baseline.
